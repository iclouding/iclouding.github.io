使用filebeats收集Docker日志

  • TOC
    {:toc}

Docker的日志，默认是使用json-file格式，存储在磁盘上，对于容器的查看日志很不方便；业界通常做法是，将docker日志收集到ELK日志系统，收集单元，通常使用filebeats来做。

通常的应用，日志一般有两种输出：控制台输出和直接将日志记录在文件中。

  • 对于直接控制台输出的日志，可以配置不通的Docker logging driver，将日志记录在文件中，发送到后端等等
  • 对于记录在日志文件中的日志，Docker官网推荐了一种方式：将日志文件链接到/dev/stdout输出，
    • nginx的日志，就将/var/log/nginx/access.log链接到了/dev/stdout，将/var/log/nginx/error.log链接到了/dev/stderr。
    • httpd的日志，将输出重定向到了/proc/self/fd/1和/proc/self/fd/2

其中， --log-opt max-size=10m --log-opt max-file=512 表示每个日志文件最多10m，最多轮询512个日志文件。--log-opt labels=containerName,tag --log-opt env=os,customer支持自定义tag信息，默认要给一个containerName，否则kibana暂时的时候，只能看到容器ID。对于tag，os，customer字段为保留字段，未来有新的需求可以扩展。

==注意：目前系统会过滤掉没有containerName的容器日志，所以如果需要暂时，需要指定该字段名称==
