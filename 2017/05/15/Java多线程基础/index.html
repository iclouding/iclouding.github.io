进程与线程

进程可以容纳多个线程

线程是程序执行的最小单位，可以说是轻量级进程。

使用多线程而不是多进程的原因是：线程之间的切换和调度比进程的成本小很多。

线程基本操作

新建线程

两种方式：

  • 继承一个Thread类
  • 实现Runnable接口

线程中断

  • stop方法

stop()方法可以立即终止一个线程，但是可能会引发数据不一致的问题。比如多个线程同时写一个对象，可能写一半时被终止而引发数据不一致。

  • interrupt()方法

interrupt()是一个实例方法，目的是通知目标线程中断，设置该线程中断的标志位。

Thread.isInterrupt()用来判断当前线程是否被中断。

Thread.interrupted()判断当前线程的中断状态，判断结束后会清除当前线程的中断标志位状态。

通知(notify)与等待(wait)

  • 等待(wait)
    • 只有锁对象，才能调用wait方法，并且必须包含在synchronized代码块中
    • 调用之后，会进入以锁对象为标识的线程池等待
  • 通知(notify)
    • 只有锁对象，才能调用notify方法，并且必须包含在synchronized代码块中
    • notify方法调用后，会从以锁对象为标识的线程池中随机唤醒一个等待线程
    • notifyAll()方法将唤醒该锁对象标识的线程池中所有等待线程

线程挂起（已废弃）

suspend挂起，被调用后线程将会被挂起，不会释放锁。

resume，将挂起的线程继续执行。

如果suspend之前发生了resume，可能会引发死锁

谦让(yield)

使当前线程让出CPU执行权。但是，让出CPU并不表示当前线程不执行了，当前线程让出CPU后，还会继续争夺CPU执行权。

等待线程结束(join)

join可以等待线程结束，有两个方法

第一个join是无限等待，会阻塞当前线程，直到目标线程执行完毕

第二个join是给出一个最大等待时间，如果超过给定的最大等待时间，会继续往下执行。

join方法底层会调用wait(0)进行等待，

当线程执行完毕后，被等待的线程会在退出前调用notifyAll()方法通知所有等待的线程继续执行。因此，尽量不要在程序中使用notify或者wait方法，以免影响其他系统API。

守护进程

守护进程是在后台默默完成一些系统性服务。比如垃圾回收

设置守护线程，必须是在线程start()方法调用之前。否则一旦线程启动，将无法改变状态。

线程优先级

使用1到10来表示线程的优先级，数字越大，级别越高。获得CPU执行权的机会越大。

内置了三个静态标量表示。

也可以自己设置

线程同步

synchronized 用来实现线程的同步。使得每一次都只有一个线程进入同步块，保证线程的安全性。

用法：

  • 使用指定锁对象：对于使用指定的锁对象，进入之前要获得给定对象的锁
  • 直接作用于实例方法：进入同步代码块之前，要获得当前对象的锁
  • 直接作用于静态方法：进入同步代码块之前，要获得当前类的锁

内存模型

原子性

原子性是指一个操作是不可中断的，即使多线程一起执行，一个操作一旦开始，就不会被其他线程所干扰。

有序性

在并发执行时，程序可能会出现乱序。

一条指令在执行的时候会分很多步骤

  • ​

可见性

可见性是指：当一个线程修改了某个共享变量的值，其他线程能否立即知道这个修改

Happen-Before原则

  • 程序顺序原则：一个线程内，保证语义的串行性
  • volatile规则：volatile变量的写，先发生于读，保证了volatile变量的可见性
  • 锁规则：解锁(unlock)必然发生在随后的加锁前
  • 传递性：A先于B，B先于C，那么A必然先于C
  • 线程的start()方法先于它的每一个动作
  • 线程的所有操作先于线程的终结(Thread.join())
  • 线程的中断，先于被中断线程的代码
  • 线程的构造函数的执行，结束先于finalize()方法
